# tagOnPlayer

![Анимация2](https://github.com/user-attachments/assets/ec4425f7-baf5-4470-ac13-84a490e06d0b)

ASI-плагин для **GTA San Andreas 1.0 US** + **SA:MP** (`samp.dll`, Win32): после штатных циклов **`CPlayerTags`** дорисовывает над **локальным** игроком те же элементы, что клиент рисует для удалённых:

- **`CPlayerTags::DrawLabel`** — текст вида **`Nick (ID)`**
- **`CPlayerTags::DrawHealthBar`** — полоски HP и брони

Опционально — **зеркало исходящего чата** в **`CChatBubble`** (см. ниже).

Исходники и релизы: **[dmitriyewich/tagOnPlayer](https://github.com/dmitriyewich/tagOnPlayer)**.

---

## Требования

- Игра: **GTA SA 1.0 US**.
- Клиент: **`samp.dll`** x86; версия определяется по **RVA entry point** модуля. Поддерживаются строки таблицы **`kSupportedVersions`** в **`source/main.cpp`** (имена в UI: **R1**, **R2**, **R3**, **R3-1**, **R4**, **R4-2**, **R5-1**, **DL-R1**). Сборки с тем же EP, что **R5-1** или **DL-R1**, используют ту же строку (например R5-2, DL-R1-2).
- Если EP не распознан, плагин **не ставит хуки**.

**Канон всех RVA/оффсетов** — только таблица **`kSupportedVersions`** в коде; в README дублировать числа не нужно.

---

## Установка

1. Собрать или взять готовый **`tagOnPlayer.asi`** (см. [Сборка](#сборка)).
2. Положить **`tagOnPlayer.asi`** в каталог с **`gta_sa.exe`** (рядом с **`samp.dll`**, как принято для ASI).
3. Рядом с **`tagOnPlayer.asi`** создаётся/используется **`tagOnPlayer.ini`** (секция **`[Settings]`**).

---

## Конфиг `tagOnPlayer.ini`, секция `[Settings]`

| Ключ | По умолчанию | Описание |
|------|----------------|----------|
| **`Command`** | `/tagon` | Команда в чат SA:MP для переключения дорисовки тега/полос (без отправки на сервер). |
| **`EnabledByDefault`** | `1` | `1` — дорисовка включена при старте; `0` — выключена до первого переключения командой. |
| **`MirrorOwnChatBubble`** | `0` | `1` — дублировать свой обычный текст чата в **`CChatBubble`** (как «бабл» над головой). |
| **`ChatBubbleLifeMs`** | `6000` | Длительность бабла для зеркала (мс), по смыслу как у серверного чат-бабла; допустимый диапазон **500–600000**. |

Пустой **`Command`** в INI подменяется значением по умолчанию; ключи при старте перезаписываются актуальными строками (как в коде).

## Зеркало чата (`MirrorOwnChatBubble=1`)

Включается только при **`MirrorOwnChatBubble=1`**. Тогда:

1. **`CChatBubble::Add`** — текст из исходящего чата передаётся в бабл (через MinHook на функцию по RVA **`localPlayerChatOffset`** в таблице версии — штатный путь вывода сообщения в чат для локального игрока). Если для версии **`localPlayerChatOffset == 0`** (в текущей таблице так у **R2**), этот путь **не подключается**.
2. **`CChatBubble::Draw`** в **`samp.dll`** — два вида правок исполняемого кода (без отдельного коммита в репозитории игры):
   - **Обход «пустого слота» для локального ID** — в цикле отрисовки баблов указатель из пула для слота локального игрока бывает нулевым; подставляется локальный игрок и выполнение продолжается (**трамплин**). Вариант зависит от клиента: короткий патч **8** или **12** байт (см. **`chatBubblePoolNullTrampKind`** в **`kSupportedVersions`**).
   - **Пропуск near-`je`** после проверки флага видимости слота (**6×NOP** на **`0F 84 …`**) — чтобы штатный код не отбрасывал итерацию для локального игрока с нулевым флагом.

Срок отображения зеркала — **`ChatBubbleLifeMs`**.

## Сборка

- Конфигурация: **`Release`**, платформа: **`Win32`**.
- Из корня репозитория (путь к MSBuild при необходимости поправить под установку VS):

```text
"C:\Program Files\Microsoft Visual Studio\18\Community\MSBuild\Current\Bin\MSBuild.exe" tagOnPlayer.sln /p:Configuration=Release /p:Platform=Win32
```

- Артефакт: **`build\Release\Win32\tagOnPlayer.asi`** (как задано в **`tagOnPlayer.vcxproj`**).

В репозитории на GitHub Actions см. **`.github/workflows/build-release-win32.yml`** (на раннере задаётся **`PlatformToolset`** для CI).

## Прочее

- Переключение дорисовки тега/полос: команда из **`Command`** (по умолчанию **`/tagon`**), обрабатывается детуром **`SendCommand`**.
- Хуки отрисовки: **`CPlayerTags`** — циклы меток и полос HP (MinHook + собственный трамплин на 9 байт инструкции, как в проекте).
