# tagOnPlayer

<img width="480" height="270" alt="tagOnPlayer" src="https://github.com/user-attachments/assets/ec4425f7-baf5-4470-ac13-84a490e06d0b" />

ASI-плагин для **GTA San Andreas 1.0 US** + **SA:MP** (`samp.dll`, Win32): после штатных циклов **`CPlayerTags`** дорисовывает над **локальным** игроком те же элементы, что клиент рисует для удалённых — **`DrawLabel`** (ник и ID) и **`DrawHealthBar`**. Опционально — зеркало исходящего чата в **`CChatBubble`**.

**Исходники и релизы:** [dmitriyewich/tagOnPlayer](https://github.com/dmitriyewich/tagOnPlayer)

---

## Возможности

- Дорисовка тега **`Nick (ID)`** и полос **HP / броня** над локальным игроком (тот же API, что для удалённых).
- Переключение дорисовки командой в чат SA:MP (по умолчанию **`/tagon`**, настраивается в INI).
- **Зеркало чата** (`MirrorOwnChatBubble=1`): **`CChatBubble::Add`** + те же правки **`CChatBubble::Draw`**, что ниже (только хук на чат и зеркалирование строки).
- **Текст по команде** (секция **`[OverlayCommands]`**): тот же путь, что зеркало — **`CChatBubble::Add`** с цветом из правила и **`ChatBubbleLifeMs`**; хвост после команды в бабл; пустая команда — короткий **`Add`** для гашения. Патчи **`Draw`** включаются, если **`MirrorOwnChatBubble=1`** **или** в INI задан **`Count` > 0** для оверлея.

## Требования и ограничения

| Параметр | Значение |
|----------|----------|
| Игра | GTA San Andreas **1.0 US** |
| Архитектура | **Win32 (x86)** |
| SA:MP | Поддерживаемые `samp.dll` (по entry point): **R1, R2, R3, R3-1, R4, R4-2, R5-1, DL-R1** (таблица **`kSupportedVersions`** в **`source/main.cpp`**) |
| Неизвестный клиент | Хуки **не** ставятся |
| Ограничение | Не заменяйте **`tagOnPlayer.asi`** при запущенной игре |

**Канон RVA/оффсетов** — только код; в README числа не дублируются.

## Установка

1. Получите **`tagOnPlayer.asi`** ([релиз](https://github.com/dmitriyewich/tagOnPlayer/releases) или локальная [сборка](#сборка)).
2. Поместите **`tagOnPlayer.asi`** в каталог с **`gta_sa.exe`** (рядом с **`samp.dll`**, как принято для ASI).
3. Если рядом с ASI **нет** **`tagOnPlayer.ini`**, при первом чтении конфига создаётся файл с дефолтным содержимым (литерал в **`source/main.cpp`**, **`WriteDefaultTagOnPlayerIniIfAbsent`**). Если файл уже есть, плагин не перезаписывает его целиком; секция **`[Settings]`** при старте по-прежнему перезаписывается актуальными значениями.

## Настройки (`tagOnPlayer.ini`, секция `[Settings]`)

| Ключ | По умолчанию | Описание |
|------|----------------|----------|
| **`Command`** | `/tagon` | Команда в чат для вкл/выкл дорисовки (на сервер не уходит). |
| **`EnabledByDefault`** | `1` | `1` — дорисовка включена при старте; `0` — выкл. до первой команды. |
| **`MirrorOwnChatBubble`** | `1` | `1` — дублировать свой текст чата в бабл над головой; **`0`** — выкл. Если ключа в INI нет, подставляется **`1`**. |
| **`ChatBubbleLifeMs`** | `6000` | Длительность зеркала (мс), диапазон **500–600000**. |
| **`OverlayBubbleLinePx`** | `257` | Для **`BubbleTemplate`**: макс. ширина строки бабла (px) при симуляции переноса; **`0`** — не вставлять **`\\n`** перед **`{RRGGBB}`**. Дефолт **257** совпадает со всеми поддерживаемыми **`samp.dll`** (rizin: **`push 0x101`** перед внутренним **`DrawText`**; см. **`sampBubbleDrawTextLinePx`** в **`kSupportedVersions`**). Подстройте, если на клиенте перенос расходится. |

Пустой **`Command`** в файле заменяется значением по умолчанию; при старте ключи **`[Settings]`** перезаписываются актуальными строками. Секция **`[OverlayCommands]`** плагин не меняет.

### Текст над головой по команде (`[OverlayCommands]`)

До **16** правил. Сравнение с вводом — по **первому токену** без учёта регистра (после команды нужен пробел или конец строки, чтобы **`/ame`** не совпало с **`/american`**).

На каждое правило задаются **`CmdN`**, **`ColorN`**, **`ForwardN`**; общий счётчик — **`Count`**. Без **`BubbleTemplateN`** хвост после команды (**`/me привет`** → **`привет`**) передаётся в бабл без изменений, цвет — из **`ColorN`**. **`BubbleTemplate`**, **`BubbleSplit`**, **`AccentColor`** — см. ниже.

| Ключ | Описание |
|------|-----------|
| **`Count`** | Сколько правил читать: `1…N` (пусто или **`0`** — оверлей по командам выключен). |
| **`CmdN`** | Строка команды, например **`/me`**. |
| **`ColorN`** | Цвет для **`CChatBubble::Add`** (см. форматы ниже). |
| **`ForwardN`** | **`1`** (по умолчанию) — сначала штатный **`SendCommand`**, потом наш бабл (иначе клиент перезапишет цвет, напр. для **`/me`**); **`0`** — не слать строку на сервер. |

**Форматы `ColorN`** (dword **D3DCOLOR** в порядке **AARRGGBB**; см. [SA:MP wiki](https://sampwiki.blast.hk/wiki/Color_list) / [open.mp](https://open.mp/docs/scripting/resources/colorslist)):

- Целое **со знаком в десятичной записи**, например **`-6684673`** — то же 32-битное значение, что в скриптах.
- **`{RRGGBB}`** — шесть hex-цифр: красный, зелёный, синий; альфа в dword задаётся как **`FF`**.
- **`{AARRGGBB}`** — восемь hex-цифр: полный dword **D3DCOLOR** (альфа, красный, зелёный, синий).

Запись в INI — в **каноническом** порядке **AARRGGBB** (как у тегов и в wiki); перед **`CChatBubble::Add`** плагин перекодирует dword: у клиента разбор цвета бабла не совпадает с прямым **AARRGGBB** для **`DrawLabel`**.

#### Шаблоны в бабле

При непустом **`BubbleTemplateN`** строка бабла собирается по шаблону (префикс **`*`**, два цвета и т.д.). Плейсхолдеры:

| Фрагмент | Замена |
|----------|--------|
| **`{0}`** | Первый сегмент хвоста (до первого **`BubbleSplitN`** или весь хвост, если split пуст). |
| **`{1}`** | Второй сегмент (после первого split); пусто, если разделителя не было. |
| **`{c1}`** | Вставка **`{RRGGBB}`** по **`ColorN`** (для смены цвета в строке, по аналогии с чатом). |
| **`{c2}`** | Вставка **`{RRGGBB}`** по **`AccentColorN`** или, если акцента нет, по **`ColorN`**. |

Остальные символы в шаблоне копируются как есть (пробелы, `*`, текст). В итоговую строку подставляются литералы **`{RRGGBB}`**; несколько цветов в одном бабле зависят от клиента SA:MP. Если вставки **`{RRGGBB}`** не разбираются, используется базовый цвет из **`ColorN`** в **`Add`**. Штатный **`ID3DXFont::DrawText`** может разорвать код **`{RRGGBB}`** между строками. При **`OverlayBubbleLinePx` > 0** плагин симулирует ту же ширину строки посимвольно (GDI **Arial** bold ≈ D3DX на поддерживаемых клиентах) и вставляет **`\\n`** перед **`{c1}`** / **`{c2}`** **только если** иначе токен цвета оказался бы разрезан посередине (без лишних переносов на каждый цвет).

| Ключ | Описание |
|------|-----------|
| **`BubbleTemplateN`** | Шаблон; **пусто** — поведение по умолчанию (сырой хвост). Итог не длиннее **256** символов. |
| **`BubbleSplitN`** | Подстрока во вводе (первое вхождение): делит хвост на **`{0}`** и **`{1}`**. Пусто — весь хвост = **`{0}`**. |
| **`AccentColorN`** | Второй цвет для **`{c2}`**; без ключа **`{c2}`** совпадает с **`{c1}`**. |

Команда **без текста** (только токен или пробелы) — сброс бабла через **`Add`** с пустой строкой и минимальным сроком (как при зеркале — поведение клиента SA:MP).

**Пример без шаблонов** (хвост команды в бабл как в чате):

```ini
[OverlayCommands]
Count=2
Cmd1=/me
Color1={FFC2A2}
Forward1=1
Cmd2=/do
Color2={D6A2E8}
Forward2=1
```

**Пример с шаблонами:**

```ini
[OverlayCommands]
Count=2
Cmd1=/me
Color1={FFC2A2}
Forward1=1
BubbleTemplate1={c1}* {0}
Cmd2=/do
Color2={ff99ff}
Forward2=0
```

**Два сегмента** (ввод **`/todo привет*помахав рукой`** — разделитель **`*`** в тексте):

```ini
[OverlayCommands]
Count=1
Cmd1=/todo
Color1={C8C8C8}
AccentColor1={FF99FF}
Forward1=1
BubbleSplit1=*
BubbleTemplate1={c1}{0}*{c2}{1}
```

### Зеркало чата и локальный бабл (`CChatBubble`)

1. **MinHook** на **`localPlayerChatOffset`** — только при **`MirrorOwnChatBubble=1`** (исходящий текст чата без ведущего **`/`**). Цвет бабла зеркала — **белый** (**`0xFFFFFFFF`**, непрозрачный ARGB). Если оффсет **`0`** (у **R2** в таблице), хук не ставится.
2. При **`MirrorOwnChatBubble=1`** **или** при **`[OverlayCommands]`** с **`Count` > 0**: в **`CChatBubble::Draw`** — трамплин пустого слота пула для локального ID (патч **8** или **12** байт) и **6×NOP** на near-**`je`** у проверки видимости слота — иначе локальный **`Add`** не рисуется.

- Visual Studio, конфигурация **`Release`**, платформа **`Win32`**.
