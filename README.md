# tagOnPlayer

<img width="480" height="270" alt="tagOnPlayer" src="https://github.com/user-attachments/assets/ec4425f7-baf5-4470-ac13-84a490e06d0b" />

ASI-плагин для **GTA San Andreas 1.0 US** + **SA:MP** (`samp.dll`, Win32): после штатных циклов **`CPlayerTags`** дорисовывает над **локальным** игроком те же элементы, что клиент рисует для удалённых — **`DrawLabel`** (ник и ID) и **`DrawHealthBar`**. Опционально — зеркало исходящего чата в **`CChatBubble`**.

**Исходники и релизы:** [dmitriyewich/tagOnPlayer](https://github.com/dmitriyewich/tagOnPlayer)

---

## Возможности

- Дорисовка тега **`Nick (ID)`** и полос **HP / броня** над локальным игроком (тот же API, что для удалённых).
- Переключение дорисовки командой в чат SA:MP (по умолчанию **`/tagon`**, настраивается в INI).
- **Зеркало чата** (`MirrorOwnChatBubble=1`): **`CChatBubble::Add`** + те же правки **`CChatBubble::Draw`**, что ниже (только хук на чат и зеркалирование строки).
- **Стек чат-баблов** (`StackChatBubbles=1`): несколько последних **`CChatBubble`** на игрока (все ID, включая локального) рисуются **одним многострочным баблом** (старые сверху, новый снизу), цвет каждой строки переносится через **`{RRGGBB}`**. Межстрочные интервалы и перенос — родные (наложений нет в принципе); весь блок поднимается так, чтобы новейший оказался на штатной высоте бабла (через временный сдвиг глобали **`getChatBubbleHeight`**). Без ImGui. Истечение строк — по **`ChatBubbleLifeMs`**/`lifeSpan`.
- **Текст по команде** (секция **`[OverlayCommands]`**): тот же путь, что зеркало — **`CChatBubble::Add`** с цветом из правила и **`ChatBubbleLifeMs`**; хвост после команды в бабл; пустая команда — короткий **`Add`** для гашения. Патчи **`Draw`** включаются, если **`MirrorOwnChatBubble=1`**, **`StackChatBubbles=1`** **или** в INI задан **`Count` > 0** для оверлея.

## Требования и ограничения

| Параметр | Значение |
|----------|----------|
| Игра | GTA San Andreas **1.0 US** |
| Архитектура | **Win32 (x86)** |
| SA:MP | Поддерживаемые `samp.dll` (по entry point): **R1, R2, R3, R3-1, R4, R4-2, R5-1, DL-R1** (таблица **`kSupportedVersions`** в **`source/main.cpp`**) |
| Неизвестный клиент | Хуки **не** ставятся |
| Ограничение | Не заменяйте **`tagOnPlayer.asi`** при запущенной игре |

**Канон RVA/оффсетов** — только код; в README числа не дублируются.

## Установка

1. Получите **`tagOnPlayer.asi`** ([релиз](https://github.com/dmitriyewich/tagOnPlayer/releases) или локальная [сборка](#сборка)).
2. Поместите **`tagOnPlayer.asi`** в каталог с **`gta_sa.exe`** (рядом с **`samp.dll`**, как принято для ASI).
3. Если рядом с ASI **нет** **`tagOnPlayer.ini`**, при первом чтении конфига создаётся файл с дефолтным содержимым (литерал в **`source/main.cpp`**, **`WriteDefaultTagOnPlayerIniIfAbsent`**). Если файл уже есть, плагин не перезаписывает его целиком; секция **`[Settings]`** при старте по-прежнему перезаписывается актуальными значениями.

## Настройки (`tagOnPlayer.ini`, секция `[Settings]`)

| Ключ | По умолчанию | Описание |
|------|----------------|----------|
| **`Command`** | `/tagon` | Команда в чат для вкл/выкл дорисовки (на сервер не уходит). |
| **`EnabledByDefault`** | `1` | `1` — дорисовка включена при старте; `0` — выкл. до первой команды. |
| **`MirrorOwnChatBubble`** | `1` | `1` — дублировать свой текст чата в бабл над головой; **`0`** — выкл. Если ключа в INI нет, подставляется **`1`**. |
| **`ChatBubbleLifeMs`** | `6000` | Длительность зеркала (мс), диапазон **500–600000**. |
| **`OwnChatBubbleColor`** | `{FFFFFF}` | Цвет текста зеркала для исходящего чата **без** ведущего **`/`** (не команды оверлея). Те же форматы, что у **`ColorN`** (см. ниже). Пустое или неверное значение — **`{FFFFFF}`**. |
| **`OverlayBubbleLinePx`** | `257` | Для **`BubbleTemplate`**: макс. ширина строки бабла (px) при симуляции переноса; **`0`** — не вставлять **`\\n`** перед вставленным **`{RRGGBB}`**. Дефолт **257** совпадает со всеми поддерживаемыми **`samp.dll`** (rizin: **`push 0x101`** перед внутренним **`DrawText`**; см. **`sampBubbleDrawTextLinePx`** в **`kSupportedVersions`**). Подстройте, если на клиенте перенос расходится. |
| **`StackChatBubbles`** | `0` | **`1`** — хранить до **`StackedChatBubbleMax`** последних баблов на каждого игрока и рисовать столбиком (новый снизу, старые вверх) одним многострочным баблом; **`0`** — штатный один бабл на слот. |
| **`StackedChatBubbleMax`** | `5` | Глубина очереди на игрока (**1…16**). |
| **`StackedChatBubbleMarginPx`** | `5` | Доп. подъём всего блока стека над ником (`Px/14` строки сверх базового подъёма в 1 строку). Интервал между строками — родной. Диапазон **0…64**. |

Пустой **`Command`** в файле заменяется значением по умолчанию; при старте ключи **`[Settings]`** перезаписываются актуальными строками. Секция **`[OverlayCommands]`** плагин не меняет.

### Текст над головой по команде (`[OverlayCommands]`)

До **16** правил. Сравнение с вводом — по **первому токену** без учёта регистра (после команды нужен пробел или конец строки, чтобы **`/ame`** не совпало с **`/american`**).

На каждое правило задаются **`CmdN`**, **`ColorN`**, **`ForwardN`**; общий счётчик — **`Count`**. Без **`BubbleTemplateN`** хвост после команды (**`/me привет`** → **`привет`**) передаётся в бабл без изменений, цвет — из **`ColorN`**. **`BubbleTemplate`**, **`BubbleSplit`**, **`AccentColor`** — см. ниже.

| Ключ | Описание |
|------|-----------|
| **`Count`** | Сколько правил читать: `1…N` (пусто или **`0`** — оверлей по командам выключен). |
| **`CmdN`** | Строка команды, например **`/me`**. |
| **`ColorN`** | Цвет для **`CChatBubble::Add`** (см. форматы ниже). |
| **`ForwardN`** | **`1`** (по умолчанию) — сначала штатный **`SendCommand`**, потом наш бабл (иначе клиент перезапишет цвет, напр. для **`/me`**); **`0`** — не слать строку на сервер. |

**Форматы `ColorN` / `OwnChatBubbleColor` / `AccentColorN`** — dword для аргумента **`CChatBubble::Add`** в упаковке **RRGGBBAA**: старший байт **R**, затем **G**, **B**, младший **A** (как десятичное значение, например **`866792447`** ≡ **`0x33AA33FF`**).

- **`{RRGGBB}`** — шесть hex-цифр; в dword подставляется **A = `FF`** (непрозрачный).
- **`{RRGGBBAA}`** — восемь hex-цифр в том же порядке каналов (**не** `{AARRGGBB}` из старых версий плагина).
- **Голый hex** ровно **6** или **8** символов **`[0-9A-Fa-f]`** (без скобок, без пробелов внутри) — то же, что **`{RRGGBB}`** / **`{RRGGBBAA}`**.
- **Десятичное целое со знаком** — любая другая строка после **`GetPrivateProfileString`**: то же 32-битное значение, что уйдёт в **`Add`** (как в скриптах SA:MP для этого dword).

Штатный **`CChatBubble::Add`** в **`samp.dll`** перед записью в слот выполняет **`(color >> 8) | 0xFF000000`** (принудительная непрозрачность старшего байта в **сохранённом** dword отрисовки); в **тексте** бабла клиент разбирает только вставки вида **`{RRGGBB}`** (шесть hex, rizin R1: **`0x100af540`**). На **всех** строках **`kSupportedVersions`** (R1 … DL-R1) Rizin показывает ту же схему для **`chatBubbleAddOffset`** — отдельные ветки парсера INI по версии **не** нужны.

**Миграция с версий плагина до 2026-05:** старый формат **`{AARRGGBB}`** (в скобках шли **AA RR GG BB**) **не** читается. Новая восьмёрка **`{RRGGBBAA}`** — **RR GG BB AA** (те же байты каналов, другой порядок в строке). Пример: было **`{AARRGGBB}`** → стало **`{RRGGBBAA}`**, где **RR GG BB** — это бывшие **RR GG BB** (цифры 3–8 старой записи), а **AA** — бывшие первые два hex (**AA**). Шаблоны: **`{0}`** → **`[a]`**, **`{1}`** → **`[b]`**, **`{c1}`** → **`[c]`**, **`{c2}`** → **`[d]`**; при необходимости **`[msg]`** — весь хвост команды.

#### Шаблоны в бабле

При непустом **`BubbleTemplateN`** строка бабла собирается по шаблону. Плейсхолдеры (после **`[`** регистр как в таблице):

| Фрагмент | Замена |
|----------|--------|
| **`[msg]`** | Весь хвост команды (после первого токена), с trim пробелов по краям. |
| **`[a]`** | Сегмент до первого вхождения **`BubbleSplitN`**; если split пуст — весь хвост (как раньше **`{0}`**). |
| **`[b]`** | Сегмент после split; пусто, если разделителя не было (**`{1}`**). |
| **`[c]`** | Вставка литерала **`{RRGGBB}`** из **`ColorN`** (для смены цвета в строке бабла, как в чате). |
| **`[d]`** | Вставка **`{RRGGBB}`** из **`AccentColorN`**, если задан; иначе как **`[c]`**. |

Остальные символы копируются как есть. Если **`[`** не начинает известный токен, символ **`[`** копируется литералом. В итоговую строку для вставок **`[c]`** / **`[d]`** подставляются только **`{RRGGBB}`** (штатный клиент в тексте бабла **8** hex в скобках не разбирает). При **`OverlayBubbleLinePx` > 0** плагин вставляет **`\\n`** перед вставкой **`{RRGGBB}`** **только если** иначе токен оказался бы разрезан между строками.

| Ключ | Описание |
|------|-----------|
| **`BubbleTemplateN`** | Шаблон; **пусто** — поведение по умолчанию (сырой хвост). Итог не длиннее **256** символов. |
| **`BubbleSplitN`** | Подстрока во вводе (первое вхождение): делит хвост на **`[a]`** и **`[b]`**. Пусто — весь хвост = **`[a]`**. |
| **`AccentColorN`** | Второй цвет для **`[d]`**; без ключа **`[d]`** совпадает с **`[c]`**. |

Команда **без текста** (только токен или пробелы) — сброс бабла через **`Add`** с пустой строкой и минимальным сроком (как при зеркале — поведение клиента SA:MP).

**Пример без шаблонов** (хвост команды в бабл как в чате):

```ini
[OverlayCommands]
Count=2
Cmd1=/me
Color1={FFC2A2}
Forward1=1
Cmd2=/do
Color2={D6A2E8}
Forward2=1
```

**Пример с шаблонами:**

```ini
[OverlayCommands]
Count=2
Cmd1=/me
Color1={FFC2A2}
Forward1=1
BubbleTemplate1=[c]* [a]
Cmd2=/do
Color2={ff99ff}
Forward2=0
```

**Два сегмента** (ввод **`/todo привет*помахав рукой`** — разделитель **`*`** в тексте):

```ini
[OverlayCommands]
Count=1
Cmd1=/todo
Color1={C8C8C8}
AccentColor1={FF99FF}
Forward1=1
BubbleSplit1=*
BubbleTemplate1=[c][a]*[d][b]
```

### Зеркало чата и локальный бабл (`CChatBubble`)

1. **MinHook** на **`localPlayerChatOffset`** — только при **`MirrorOwnChatBubble=1`** (исходящий текст чата без ведущего **`/`**). Цвет бабла зеркала — из **`OwnChatBubbleColor`** (по умолчанию белый **`{FFFFFF}`**). Если оффсет **`0`** (у **R2** в таблице), хук не ставится. **ID игрока `0` в SA:MP допустим** — бабл и дорисовка DL используют тот же слот, что и для остальных ID (`< 1004`).
2. При **`MirrorOwnChatBubble=1`** **или** при **`[OverlayCommands]`** с **`Count` > 0**: в **`CChatBubble::Draw`** — трамплин локального ID до remote-player chain (патч **8** или **11** байт), SEH-валидация remote-chain перед возвратом в оригинальный код и **6×NOP** на near-**`je`** у проверки видимости слота — иначе локальный **`Add`** не рисуется. Для **DL-R1** локальный слот идёт дальше по штатному пути **`CChatBubble::Draw`** с `CPed*` локального игрока даже когда слот бабла уже заполнен; дополнительно guarded unsafe remote-chain deref на **`+0x66A03`** / **`+0x66A0A`**. Отдельный fallback через **`CLabel::Draw`** не используется.
